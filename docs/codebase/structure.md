
## What is the goal of easysort?
Easysort is meant to be a complete system for setting up a system to analyze waste streams in real time and host results.

This could be:
- A simple phone sending information to a certain cloud endpoint
- Downloading images, analyzing them in realtime, uploading results to cloud endpoint
- Easytrack (web application) for visualizing results and hosting a dashboard for the waste facility.
- Exposing an API for live systems at recycling centers

## How should the codebase be?
- Lots of regression tests
- Ultra simple, so structural issues can actually be spotted.
- Line-count to be as low as possible (higher focus on readability)
- As low number of dependencies as possible
